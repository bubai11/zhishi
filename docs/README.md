# 文档治理说明

本目录用于区分当前可信文档和历史归档资料，避免多个 AI 或阶段性总结互相冲突。

## 当前可信入口

优先阅读以下文件：

- `../README.md`：项目真实技术栈、目录结构和维护约定。
- `../QUICKSTART.md`：本地快速启动。
- `../SETUP.md`：完整安装与环境配置，仍需后续校准。
- `../backend/DEVELOPMENT.md`：后端开发说明。
- `../backend/DATA_IMPORT_GUIDE.md`：WCVP、IUCN、WDPA 等数据导入流程。
- `../backend/scripts/README.md`：数据脚本分层、官方入口和治理规则。

## 历史归档

`docs/archive/` 保存历史报告、阶段性总结、旧方案和 AI 生成文档。归档文件只作为背景材料，不作为当前实现的依据。

建议分类：

- `docs/archive/ai-generated/`：AI 生成的总结、检查清单、白皮书、技术总结。
- `docs/archive/reports/`：数据完整性报告、交付报告、历史审查材料。
- `docs/archive/old-plans/`：旧的前端重构方案、数据补全方案、命名映射方案。

## 维护规则

- 根目录尽量只保留启动、安装、数据库初始化和当前可信入口文档。
- 新增长期文档前，先判断是否应合并到现有入口文档。
- 阶段性报告默认进入 `docs/archive/`。
- 如果文档描述和代码不一致，以代码和当前可信入口为准，并及时修正文档。
