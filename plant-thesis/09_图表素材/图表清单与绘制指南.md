### 图3.1 系统用例图

**工具**：Draw.io（diagrams.net）
**保存为**：`图3.1_系统用例图.drawio` + 导出 `图3.1_系统用例图.png`

**绘制要点**：
```
参与者（Actor，小人图标）：
  ├── 游客（Guest）
  ├── 注册用户（User）—— 继承游客（用泛化箭头连接）
  └── 管理员/数据工程师（Admin）—— 独立Actor

游客用例（椭圆）：
  ├── 浏览植物库
  ├── 关键词搜索植物
  ├── 查看植物详情
  ├── 浏览分类树
  └── 查看分布热力图与统计图

注册用户追加用例（在游客基础上）：
  ├── 注册 / 登录
  ├── 收藏植物
  ├── 查看浏览历史
  ├── 参与学习测验
  ├── 查看成就
  ├── 查看濒危预警
  └── 标记预警已读/忽略

管理员用例：
  ├── 导入WCVP数据
  ├── 补全中文名
  └── 管理图片资源

系统边界：画一个大矩形，所有用例椭圆在矩形内，Actor在外
```

### 图4.4 核心数据库ER图

**工具**：**MySQL Workbench**（推荐自动逆向生成）

**操作步骤**：
1. 打开 MySQL Workbench，连接本地数据库
2. 菜单：Database → Reverse Engineer
3. 选择 `plant_knowledge` 数据库，点击执行
4. 生成的EER图中，选中以下核心表（可隐藏其余表）：
   `taxa`, `plants`, `plant_detail`, `plant_distributions`, `threatened_species`, `media_assets`, `plant_media`, `user`, `favorites`, `browse_events`, `quizzes`, `questions`, `question_options`, `quiz_attempts`, `attempt_answers`
5. 导出为PNG图片

如果MySQL Workbench不便使用，也可以用Draw.io手绘以下核心关系：
```
taxa (自关联: parent_id → id)
  ↓ 1:N
plants
  ├── 1:1 → plant_detail
  ├── 1:N → plant_distributions → wgsrpd_regions
  ├── 1:N → threatened_species
  ├── M:N → media_assets (通过plant_media)
  ├── 1:N → favorites → user
  └── 1:N → browse_events → user

user
  ├── 1:N → favorites
  ├── 1:N → browse_events
  └── 1:N → quiz_attempts
              └── 1:N → attempt_answers

quizzes
  └── 1:N → questions
              └── 1:N → question_options
```

### 图5.1 中文名四级优先级映射流程图

**工具**：Draw.io
**保存为**：`图5.1_中文名映射流程图.drawio` + 导出PNG

**绘制要点**（纵向判断流程图）：
```
[开始：处理一个植物学名]
    ↓
[第1级：查询iPlant数据库] → 有结果? → 是 → [写入中文名，标记来源=iPlant] → [结束]
    ↓ 否
[第2级：查询GBIF API] → 命中且置信度>0.9? → 是 → [写入中文名，标记来源=GBIF] → [结束]
    ↓ 否
[第3级：查询Wikidata] → 有中文标签? → 是 → [写入中文名，标记来源=Wikidata] → [结束]
    ↓ 否
[第4级：学名模板推断] → 可构造候选名? → 是 → [写入候选名，标记来源=template，置信度低] → [结束]
    ↓ 否
[加入人工审核队列] → [人工确认] → [写入中文名] → [结束]
```
