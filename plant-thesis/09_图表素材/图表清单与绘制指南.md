## 二、结构图文字版绘图结构（AI绘图提示词）

### 【图4.1 系统总体架构图】

**绘图提示词 / 结构**：
此图必须严格采用横向或纵向分层结构，从上到下分为：展示层、API接入层、业务逻辑层、数据访问层、核心数据层。并在侧边以“虚线框”独立标出“离线数据工程管道”，切忌将其与在线服务混为一谈。

```text
[展示层] (React 19, TypeScript, Vite, Tailwind CSS 4)
- 页面容器 (App.tsx 状态驱动路由)
- 核心组件 (植物库、分类树、可视化分析、学习中心)

      ↓ HTTP/REST API

[API接入层] (Express 4.x)
- CORS & JSON 中间件
- JWT 鉴权中间件
- 13个业务路由 (routes/)

      ↓

[业务逻辑层] (Node.js Services)
- plantService (植物展示)
- searchService (复杂权重搜索)
- taxaService (分类树懒加载)
- wcvpAnalyticsService (可视化聚合)

      ↓ ORM / Raw SQL

[数据访问层] (Sequelize ORM)
- Models & Associations

      ↓

[核心数据层] (MySQL 8.0)
- taxa, plants, plant_detail, threatened_species 等 29 张表

=== 虚线分隔 (离线环节) ===

[离线数据工程管道] (Node.js Scripts)
- 多源数据导入 (WCVP, IUCN, WDPA)
- 空间数据映射 (WGSRPD -> ISO3)
- 名称增强与补全 (iPlant, GBIF)
```

### 【图4.2 数据工程管道流程图】

**绘图提示词 / 结构**：
重点展现异构数据如何一步步变成可用的结构化数据。

```text
[外部权威数据源]
  ├── WCVP (分类学骨架)
  ├── IUCN Red List (濒危等级)
  ├── WDPA & WGSRPD (保护地与分布区域)
  └── iPlant/GBIF (中文科普描述)

      ↓

[清洗与清洗脚本层]
  ├── 导入与去重 (import-wcvp.js)
  ├── 中文名四级优先级映射 (batch-update-chinese-names.js)
  ├── 空间区域对齐映射脚本
  └── 详情与图片补全脚本

      ↓

[数据库持久化]
  └── MySQL 核心业务表
```

### 【图4.3 系统功能模块结构图】

经典的树状图，自顶向下。

### 【图4.8 核心数据库ER图】

**绘图提示词 / 结构**：
用 Crow's Foot 表示法（1:N）展示核心表关系。

```text
taxa
  └── 1:N (自关联 parent_id) → taxa
  └── 1:N → plants

plants
  ├── 1:1 → plant_detail (扩展字段)
  ├── 1:N → plant_distributions (关联 wgsrpd_regions)
  ├── 1:N → threatened_species (IUCN 数据)
  └── 1:N → plant_media (关联 media_assets)

user
  ├── 1:N → favorites (收藏 plants)
  ├── 1:N → browse_events (浏览 plants)
  └── 1:N → quiz_attempts (答题记录)

quizzes
  └── 1:N → questions
               └── 1:N → question_options
```
