# 图4.1 系统总体架构图绘制说明

## 推荐图题

图4.1 系统总体架构图

## 放置位置

建议放在第四章“4.2 系统总体架构设计”小节之后。

## 图的核心表达

该图用于说明系统采用前后端分离架构，前端负责页面展示与交互，后端负责业务逻辑与接口服务，数据库负责植物知识、用户行为和学习记录等数据存储，多源数据与脚本维护能力为系统提供基础数据支撑。

## 绘制结构

建议采用自上而下的四层结构：

1. 前端交互层
2. 后端服务层
3. 数据访问层与数据支撑层
4. 多源数据与系统支撑层

## 每层内容

### 前端交互层

技术标注：React + TypeScript + Vite

模块包括：

1. 首页与系统导览
2. 植物库与搜索筛选
3. 植物详情展示
4. 植物分类认知
5. 可视化分析页面
6. 学习中心
7. 个人中心

### 后端服务层

技术标注：Node.js + Express

模块包括：

1. RESTful API 路由
2. 植物数据服务
3. 分类查询服务
4. 可视化分析服务
5. 用户行为记录服务
6. 测验推荐与提交服务
7. 用户认证服务

### 数据访问层

技术标注：Sequelize ORM

内容包括：

1. 模型映射与关联关系
2. 查询封装与统计计算

### 数据支撑层

技术标注：MySQL 植物知识数据库

建议体现核心数据表类型：

1. 植物信息表
2. 分类与标签表
3. 植物图片表
4. 分布与保护数据表
5. 题库与答题记录表
6. 用户行为表

这里就是数据库设计在架构图中的主要体现位置。

### 多源数据与系统支撑层

内容包括：

1. 网络采集数据
2. 开放植物数据
3. 人工整理与校核数据
4. 数据清洗、导入与维护脚本

## 箭头关系

建议画以下箭头：

1. 前端交互层 → 后端服务层：HTTP 请求 / JSON 响应
2. 后端服务层 → 数据访问层：业务调用
3. 数据访问层 → 数据支撑层：数据库读写
4. 多源数据与系统支撑层 → 数据支撑层：数据清洗、导入与更新
5. 前端模块之间可简要体现：植物库 → 植物详情 → 学习中心，可视化分析 → 植物详情

## 论文中可配套说明文字

如图4.1所示，系统整体采用前后端分离架构。前端交互层基于 React、TypeScript 和 Vite 实现，主要承担植物知识浏览、分类认知、可视化分析和学习交互等页面展示任务；后端服务层基于 Node.js 与 Express 构建，负责提供植物查询、分类查询、可视化分析、用户行为记录和测验推荐等接口服务；数据访问层通过 Sequelize ORM 完成数据库模型映射和查询封装；数据支撑层以 MySQL 为核心，存储植物基础信息、分类标签、图片资源、分布保护数据、题库记录和用户行为数据。系统底层通过网络采集数据、开放植物数据和人工整理数据进行多源融合，并借助数据清洗与导入脚本保障植物知识库的持续更新。
