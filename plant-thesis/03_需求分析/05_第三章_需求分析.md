# 第三章 系统需求分析

## 3.1 功能需求分析

本系统面向普通用户提供植物科普浏览、分类认知、可视化分析与交互学习服务，同时提供基础的数据维护与运行支撑能力。结合当前系统实现情况，可将功能需求划分为前台用户功能需求和系统支撑功能需求两个部分。

### 3.1.1 前台用户功能需求

（1）首页与系统导览需求  
系统应提供统一的首页入口，对平台定位、主要内容和核心功能进行展示，并支持用户通过导航快速进入植物库、分类认知、可视化分析、学习中心和个人中心等页面。

（2）植物知识浏览需求  
系统应支持植物列表浏览、关键词搜索、条件筛选和分页展示，使用户能够高效查找感兴趣的植物对象。列表页应突出植物名称、图片和基础分类信息，满足日常科普浏览场景。

（3）植物详情展示需求  
系统应支持植物详情页展示，包括植物中文名、学名、分类、简介、图片、分布信息、保护状态及相关拓展信息。该页面是植物知识展示的核心页面，应兼顾信息完整性和可读性。

（4）植物分类认知需求  
系统应支持按照分类层级组织植物知识，帮助用户从科、属、种等角度理解植物之间的关系。分类浏览模块应具备层级展开和结构化导航能力，使用户可以从分类视角进行植物知识探索。

（5）数据可视化分析需求  
系统应支持对植物分布、热点区域、统计结果和保护信息进行图形化展示，使用户能够从宏观角度理解植物资源特征。可视化模块不仅要展示数据结果，还应具备一定交互能力，以增强用户的理解效率和参与感。

（6）学习交互需求  
系统应支持基于浏览行为的推荐测验、测验作答、成绩反馈和复习引导等功能，使用户在浏览植物知识后能够进一步进行学习巩固，形成较完整的交互体验。

（7）个人中心需求  
系统应支持展示用户基本信息、学习统计、收藏数量、浏览情况和成就信息等内容，用于体现用户的学习状态和使用轨迹。

### 3.1.2 系统支撑功能需求

（1）用户行为记录需求  
系统应记录用户在浏览、收藏、测验等环节中的关键行为数据，为推荐测验、学习统计和系统分析提供基础支撑。

（2）数据维护需求  
由于系统底层数据包含多源采集与人工整理结果，因此系统应具备基础的数据维护与更新能力，以保证植物信息和题库内容的可持续使用。该部分属于系统支撑功能，不作为论文重点展开。

（3）运行稳定性需求  
系统应保证前后端接口可正常联通，核心页面能够稳定加载，关键业务流程能够完整闭环运行。

## 3.2 非功能需求分析

### 3.2.1 性能需求

作为面向 Web 的植物科普系统，系统在性能上应满足一般教学展示与答辩演示要求。植物列表查询、植物详情查询、分类节点加载和可视化分析接口应具备较好的响应速度，避免因数据量较大导致页面长时间等待。对于用户高频访问的浏览与查询功能，应尽量保证较平稳的响应表现。

### 3.2.2 可用性需求

系统应支持主流现代浏览器访问，并在桌面端保持良好的显示效果。界面设计应简洁清晰，导航路径应明确，用户无需复杂学习即可完成植物浏览、分类探索和数据分析等核心操作。对于论文答辩场景，系统还应具备较好的演示连贯性。

### 3.2.3 安全性需求

系统在用户认证和数据交互过程中应具备基本的安全保障能力。用户密码需要加密存储，需要鉴权的接口应进行身份验证，接口参数应具备基础校验和错误处理，避免无效请求直接写入数据库。

### 3.2.4 可扩展性需求

系统应便于后续继续扩展植物数据、题库内容、可视化页面和学习功能。在架构上，应尽量保持前后端分层清晰、数据模型结构明确、业务模块相对独立，以便后续进行功能迭代。

### 3.2.5 数据完整性需求

由于系统采用多源数据融合方式构建植物知识库，因此数据完整性尤为重要。系统应尽量保证植物基础信息、图片信息、分类信息、分布信息和题库关联信息之间的对应关系正确，减少孤立数据和错误映射对前台展示效果的影响。

## 3.3 系统模块与用例分析

### 3.3.1 系统模块划分

结合系统当前实现情况，可将系统划分为以下六个功能模块：

1. 首页与系统导览模块  
2. 植物知识浏览与详情展示模块  
3. 植物分类认知模块  
4. 植物数据可视化分析模块  
5. 学习测验交互模块  
6. 用户中心与系统支撑模块

其中，植物知识浏览、分类认知和可视化分析构成系统的主要科普展示主线；学习测验和个人中心属于增强交互和学习体验的辅助模块；基础数据维护和运行支撑属于后台支撑能力，不作为论文主体内容重点展开。

### 3.3.2 普通用户用例分析

普通用户是本系统的主要使用对象。其核心使用流程如下：

1. 进入系统首页并了解系统功能；
2. 进入植物库浏览植物列表，按名称或分类检索目标植物；
3. 查看植物详情，阅读植物介绍、图片与分布信息；
4. 进入分类模块，从结构化角度理解植物的分类关系；
5. 进入分析模块，从空间分布和统计视角理解植物资源特征；
6. 在学习中心参与推荐测验并根据结果进行复习；
7. 在个人中心查看学习统计与收藏记录。

因此，普通用户的主要用例可以概括为“浏览植物知识、理解分类结构、查看可视化分析、参与学习交互、查看个人状态”。

### 3.3.3 系统支撑用例分析

系统支撑部分主要包括数据维护、内容更新和运行支撑等职责，用于保证植物数据和学习资源的持续可用。这部分功能在系统中是必要的，但其定位属于辅助支撑，而不是论文重点研究对象。在论文撰写中，可将其简要说明为“系统具备基础的数据维护与支撑能力”。

## 3.4 本章小结

本章从功能需求和非功能需求两个层面对系统进行了分析。总体来看，系统的核心目标并不是构建一个通用管理平台，而是围绕植物科普场景，提供植物知识浏览、分类认知和数据可视化分析等主要功能，并通过学习交互提升用户参与度。上述需求分析为后续系统总体设计和功能实现提供了依据。
