## 2.1 前端相关技术

### 2.1.1 React 19

React是由Meta（原Facebook）开源维护的前端JavaScript框架，基于组件化思想和声明式UI编程范式，是目前全球使用最广泛的前端框架之一[1]。本系统采用React 19版本，其引入了并发渲染（Concurrent Rendering）机制，使得组件的渲染优先级可以根据用户交互进行动态调整，显著提升了复杂页面的响应流畅度。

React的核心设计理念是**组件化**：将用户界面拆分为相互独立、可复用的组件（Component），每个组件维护自身的状态（State）和属性（Props），通过Hooks机制实现状态逻辑的复用与分离。本系统前端共实现了9个核心页面组件（Home、Library、PlantDetail、Classification、Analysis、LearningCenter、UserProfile、Navbar、Footer）和若干子组件，各组件职责清晰，便于独立开发与测试。

### 2.1.2 TypeScript

TypeScript是微软开发的JavaScript超集，在JavaScript基础上增加了静态类型系统、接口定义、枚举等特性[2]。在大型前端工程中，TypeScript能够在编译阶段发现类型错误，减少运行时异常，并显著提升代码的可读性与可维护性。

本系统前端使用TypeScript进行开发，在`types.ts`文件中统一定义了系统所有核心数据类型，包括`PlantCard`（植物卡片）、`PlantDetailResponse`（植物详情）、`TaxonomyNode`（分类节点）、`Quiz`（测验）、`Alert`（预警）等，实现了前后端数据结构的类型安全对接。

### 2.1.3 Vite 6

Vite是由Vue.js作者尤雨溪主导开发的新一代前端构建工具[3]。相比传统的Webpack，Vite在开发模式下基于浏览器原生ES Module（ESM）进行按需编译，启动速度极快；在生产构建时则使用Rollup进行代码打包与优化。本系统使用Vite 6版本，开发时支持热模块替换（HMR），显著缩短了开发调试的反馈周期。

### 2.1.4 Tailwind CSS 4

Tailwind CSS是一个以"原子化CSS"（Utility-First CSS）为核心理念的样式框架[4]。与Bootstrap等传统组件样式库不同，Tailwind提供大量细粒度的功能类（如`flex`、`text-sm`、`bg-emerald-500`），允许开发者直接在HTML/JSX中组合类名实现复杂样式，无需编写自定义CSS文件。本系统使用Tailwind CSS 4版本，结合其JIT（即时编译）引擎，实现了响应式布局与一致性设计语言，同时保持了CSS产物体积的最小化。

### 2.1.5 SVG与自定义地图可视化

系统的全球植物分布热力图并未依赖第三方图表库，而是基于浏览器原生SVG技术自行实现。系统将WGSRPD三级地理分区的GeoJSON边界数据转换为SVG路径（`<path>`元素），通过将经纬度坐标映射为像素坐标实现地理投影，并根据各区域的物种数量动态渲染颜色深浅，最终实现了支持区域高亮、缩放视口、鼠标悬停、点击选中的交互式全球分布热力图。这种自定义实现方式避免了引入大体积可视化库，提升了页面加载性能。

## 2.2 后端相关技术

### 2.2.1 Node.js与Express框架

Node.js是基于Chrome V8引擎构建的JavaScript服务端运行时，其采用单线程异步非阻塞I/O模型，在处理大量并发请求时具有较低的内存开销和较高的吞吐量，适合构建数据密集型的API服务[5]。

Express是基于Node.js的轻量级Web应用框架，提供了路由管理、中间件机制、请求/响应封装等核心功能，是目前Node.js生态中使用最广泛的Web框架之一[6]。本系统后端采用Express 4.x版本，按照MVC（Model-View-Controller）模式组织代码：`routes/`负责URL路由分发，`controllers/`负责HTTP请求与响应处理，`services/`承载核心业务逻辑，`models/`定义数据模型。系统共实现了11个路由模块（plants、taxa、search、redlist、protected-areas、quizzes、browse-events、favorites、user、media-assets、wcvp-analytics），覆盖系统全部业务域。

### 2.2.2 Sequelize ORM

Sequelize是Node.js生态中功能最完整的ORM（对象关系映射）框架，支持MySQL、PostgreSQL、SQLite等多种关系型数据库[7]。ORM框架通过将数据库表映射为JavaScript对象，使开发者可以用面向对象的方式操作数据库，无需手写SQL语句，同时提供了模型关联（Associations）、数据迁移（Migration）等高级特性。

本系统使用Sequelize 6版本，在`models/`目录下为29张数据表各定义了对应的模型文件，并在`associations.js`中统一管理模型间的关联关系（`belongsTo`、`hasOne`、`hasMany`、`belongsToMany`）。对于复杂的统计查询和性能敏感场景，系统在Sequelize基础上混用了`sequelize.query()`原生SQL查询，兼顾了开发效率与查询性能。

### 2.2.3 JWT身份认证

JSON Web Token（JWT）是一种开放标准（RFC 7519），用于在网络应用间安全地传递声明信息[8]。JWT由Header、Payload和Signature三部分组成，服务端生成Token后下发给客户端，客户端在后续请求中将Token置于`Authorization`请求头中携带，服务端通过验证签名即可完成身份鉴权，无需在服务端保存Session状态。

本系统采用JWT实现无状态身份认证，配合bcryptjs对用户密码进行哈希存储，在注册、登录、收藏、测验等需要鉴权的接口上通过中间件统一进行Token验证，确保用户数据安全。

## 2.3 数据库相关技术

### 2.3.1 MySQL 8.0

MySQL是目前全球使用最广泛的开源关系型数据库管理系统，具有高性能、高可靠性和丰富的生态支持[9]。本系统使用MySQL 8.0版本，利用其对JSON字段类型的原生支持存储植物的扩展属性（`extra`字段），并通过InnoDB引擎的外键约束保障数据参照完整性。

### 2.3.2 数据库索引优化

系统在植物检索、分类查询等高频访问路径上进行了专项索引设计。对`plants`表的中文名、学名字段建立了前缀索引，支持`LIKE 'prefix%'`形式的快速模糊搜索；对`taxa`表的`parent_id`字段建立了索引，以优化分类树递归查询；对`plant_distributions`表的`wgsrpd_level3_code`字段建立了索引，支持按地理区域高效筛选植物分布数据。所有性能索引通过专用脚本（`ensure-performance-indexes.js`）统一管理，确保在数据迁移后自动重建。

## 2.4 多源开放数据标准

本节介绍系统所采用的四个主要权威数据源，这是本系统区别于一般植物信息系统的核心技术基础。

### 2.4.1 WCVP（世界栽培植物词汇表）

WCVP（World Checklist of Vascular Plants）由英国皇家植物园邱园（Kew Gardens）维护，是目前全球最权威的维管植物名称数据库，收录了超过1,100万个植物名称条目，涵盖约35万个接受的物种[10]。系统以WCVP CSV数据文件为分类学骨架，导入科（Family）、属（Genus）、种（Species）三级分类节点，并以WCVP接受名（accepted name）为标准学名，作为系统植物数据的权威基准。

### 2.4.2 IUCN红色名录

IUCN（国际自然保护联盟）红色名录是全球最具权威性的物种濒危状况评估体系，将物种分为灭绝（EX）、野外灭绝（EW）、极危（CR）、濒危（EN）、易危（VU）、近危（NT）、无危（LC）等级别[11]。系统接入IUCN红色名录数据，构建了`threatened_species`（濒危物种）和`redlist_alerts`（濒危预警）两张核心表，实现了濒危等级统计分析与用户级预警推送功能。

### 2.4.3 WDPA（全球保护地数据库）

WDPA（World Database on Protected Areas）是全球最全面的保护地信息数据库，收录了全球超过27万个保护地的边界、类别、管理机构等信息[12]。系统接入WDPA数据，在`protected_areas`表中存储保护地记录，并支持按国家/地区代码（ISO3）、IUCN保护类别、保护地类型等维度进行检索和统计。

### 2.4.4 WGSRPD（世界植物地理分区标准）

WGSRPD（World Geographical Scheme for Recording Plant Distributions）是由国际植物分类学协会（IAPT）制定的植物地理分布记录标准，将全球划分为约369个三级地理单元（Level-3），为植物分布数据提供了标准化的空间描述框架[13]。系统利用WGSRPD三级分区代码存储每种植物的分布数据，并结合各分区的GeoJSON边界数据，实现了全球植物分布热力图的可视化展示。
