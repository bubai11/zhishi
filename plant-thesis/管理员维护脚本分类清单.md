# 管理员维护脚本分类清单

本文档用于说明系统中与数据维护、初始化、迁移和专项修复相关的脚本分类。系统脚本主要集中在 `backend/scripts/` 与根目录 `scripts/` 中。管理员日常使用时应优先通过 `backend/package.json` 中暴露的 `npm run` 命令执行，未登记脚本默认视为历史脚本、调试脚本或一次性维护脚本。

## 一、日常或周期性维护脚本

这类脚本是管理员最可能反复使用的维护入口，主要用于小批量数据修正、索引检查、名称增强、图片补全和人工审核。

| 维护内容 | 推荐命令 | 说明 |
|---|---|---|
| 小批量人工修正植物数据 | `npm run data:import -- --target=plants --file=xxx.csv --dry-run` | 先预演，确认无误后去掉 `--dry-run` 正式导入 |
| 小批量修正濒危或保护地数据 | `npm run data:import -- --target=threatened_species --file=xxx.json --dry-run` | 支持 JSON 与 CSV |
| 小批量修正保护地数据 | `npm run data:import -- --target=protected_areas --file=xxx.csv --dry-run` | 以 `site_id` 为更新键 |
| 检查和补充数据库索引 | `npm run db:ensure-indexes` | 大批量导入后建议执行 |
| iPlant 名称增强 | `npm run iplant:enrich -- --batch=100 --rounds=1` | 用于补充中文名、俗名、异名和正名关系 |
| 真实图片补全 | `node ..\scripts\enrich-real-images.js --limit=1000 --concurrency=2 --delay=300 --placeholder-only` | 建议低并发、小批次运行，不轻易使用 `--force` |
| 常见植物图片标准化 | `npm run images:standardize:common` | 将已有图片生成标准封面、详情图和缩略图 |
| 人工审核队列导出 | `npm run human-review:export` | 用于导出待人工确认的中文名数据 |
| 高价值中文名审核准备 | `npm run cn:review-high:prepare` | 生成高优先级人工审核队列 |
| 高价值中文名审核合并 | `npm run cn:review-high:merge` | 合并人工审核后的结果 |

## 二、初始化或大版本数据更新脚本

这类脚本通常只在数据库初始化、上游数据源更新或重大数据重建时执行，不属于日常维护命令。

| 数据任务 | 推荐命令 | 使用场景 |
|---|---|---|
| WCVP 主数据导入 | `npm run import:wcvp` | 初始化或 WCVP 数据源大版本更新 |
| IUCN 红色名录导入 | `npm run import:iucn -- --dry-run` | 先预演，再正式导入 |
| WDPA 保护地导入 | `npm run import:wdpa -- --dry-run` | 先预演，再正式导入 |
| 构建 WGSRPD Level 3 映射 CSV | `npm run build:wgsrpd:level3` | 区域映射文件更新时执行 |
| 导入 WGSRPD 区域-国家映射 | `npm run data:import:wgsrpd-map -- --dry-run` | 先预演，再正式导入 |
| 同步 WCVP 字段到植物表 | `npm run wcvp:sync:plants-fields` | WCVP 重导或字段补全后执行 |
| 修复分类链 | `npm run taxonomy:repair-chain` | 分类父子关系异常时执行 |
| 补全 WCVP 分类信息 | `npm run wcvp:complete` | 初始化或分类补全阶段使用 |

## 三、数据库结构迁移脚本

这类脚本用于创建表、增加字段或升级数据库结构。管理员执行前应确认数据库已备份，尤其是涉及表结构重建的脚本。

| 迁移任务 | 命令 | 风险级别 |
|---|---|---|
| 中文名缓存与基础字段迁移 | `npm run migrate:chinese-names` | 低 |
| iPlant 增强表与异名表迁移 | `npm run migrate:iplant-enrichment` | 低 |
| 前端展示字段迁移 | `npm run migrate:frontend-fields` | 中 |
| 红色名录用户预警状态迁移 | `npm run migrate:redlist-alert-user-state` | 低 |
| 分类亚门字段迁移 | `npm run taxonomy:migrate-subphylum` | 中 |
| 2026 年 3 月大版本结构迁移 | `npm run migrate:schema:2026-03` | 高 |

其中 `migrate:schema:2026-03` 包含重建部分业务表的逻辑，只适合空库初始化或明确备份后的重建场景，不建议在已有正式数据库中随意执行。

## 四、中文名与杂交植物专项脚本

这类脚本主要用于中文名覆盖率提升、人工审核和杂交植物名称整理，属于阶段性专项维护任务。

| 专项任务 | 命令 | 说明 |
|---|---|---|
| 批量更新中文名 | `npm run cn:update` | 按批次补充中文名 |
| 远程抓取中文名 | `npm run fetch:chinese-names` | 偏历史和专项用途 |
| 推进中文名覆盖率 | `npm run wcvp:chinese:coverage` | 按覆盖率目标执行 |
| 导入中文名模板 | `npm run cn:import-template` | 用于模板化中文名补全 |
| 杂交植物审核准备 | `npm run cn:review-hybrid:prepare` | 生成杂交植物审核队列 |
| 杂交中文名初始化 | `npm run cn:review-hybrid:init` | 初始化杂交中文名 |
| 杂交中文名自动填充 | `npm run cn:review-hybrid:auto-fill` | 自动生成或补全候选名 |
| 杂交审核结果导入 | `npm run cn:review-hybrid:import-direct` | 导入人工审核后的结果 |
| 杂交中文名批量处理 | `npm run cn:review-hybrid:auto-all` | 大批量专项处理 |

## 五、不建议直接运行的历史或调试脚本

以下脚本保留在仓库中，但不应作为管理员日常入口。除非明确了解其输入、输出和风险，否则不建议直接执行。

| 脚本 | 不建议直接运行的原因 |
|---|---|
| `import-wcvp-fast.js` | WCVP 导入历史变体，容易与正式入口混淆 |
| `import-wcvp-simplified.js` | WCVP 导入历史变体，默认使用 `npm run import:wcvp` |
| `debug-foc-scrape.js` | 调试脚本 |
| `tmp-*` | 临时分析脚本 |
| `ddd.txt` | 临时文件 |
| `rebuild-plants-table.sql` | 可能重建核心表，风险高 |
| `rebuild-plants-ascii.sql` | 可能重建核心表，风险高 |
| `rebuild-plants-scientific-names.js` | 历史修复脚本 |
| `populate-plant-images.js` | 早期图片填充脚本，当前优先使用真实图片补全与标准化脚本 |
| `populate-plant-category.js` | 早期分类标签填充脚本 |
| `populate-plant-ecology.js` | 早期生态数据填充脚本 |

## 六、推荐管理员常用命令集合

管理员日常维护可优先记住以下命令：

```bash
cd backend
npm run data:import -- --target=plants --file=xxx.csv --dry-run
npm run data:import -- --target=plants --file=xxx.csv
npm run db:ensure-indexes
npm run iplant:enrich -- --batch=100 --rounds=1
npm run images:standardize:common
npm run human-review:export
```

图片补全建议单独执行：

```bash
node ..\scripts\enrich-real-images.js --limit=1000 --concurrency=2 --delay=300 --placeholder-only
```

维护原则如下：

1. 大型导入脚本只在初始化或数据源更新时执行。
2. 小批量修正优先使用 `data:import`，并先执行 `--dry-run`。
3. 图片补全应采用低并发、小批次策略，不轻易使用 `--force`。
4. iPlant 当前定位为中文名称增强与名称关系补全来源，不应描述为完整植物百科详情采集脚本。
5. 未登记到 `backend/package.json` 的脚本默认不作为管理员日常入口。
